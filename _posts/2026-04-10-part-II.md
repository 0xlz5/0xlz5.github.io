## 3 — Secure Coding vs. Code Security  
*Two terms that look similar but live in different worlds*

- [OWASP Secure Coding Practices](https://owasp.org/www-project-secure-coding-practices-quick-reference-guide/stable/en/)

Secure Coding includes:

- input validation  
- sanitization  
- safe session handling  
- cryptographic hygiene  
- avoiding dangerous patterns

## 3.3 — The philosophical split

- Secure Coding → *“Make the code safe.”*  
- Code Security → *“Make the system safe even when the code changes.”*

Secure Coding is **grammar**.  
Code Security is **syntax + context + evolution**.

You need both.
