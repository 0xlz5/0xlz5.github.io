# Part D — Secure Coding vs Code Security

Two terms that look similar but live in different worlds.

---

## D.1 — Secure Coding (OWASP)

Secure Coding is about **writing code that resists being broken**.

- input validation  
- sanitization  
- safe session handling  
- cryptographic hygiene  
- reducing attack surface  

### References

- [OWASP Secure Coding Practices](https://owasp.org/www-project-secure-coding-practices-quick-reference-guide/)  
- [CERT Coding Standards](https://wiki.sei.cmu.edu/confluence/display/seccode/SEI+CERT+Coding+Standards)

---

## D.2 — Code Security (the ecosystem)

Code Security is not about how you write code —  
it’s about **how code lives**.

- SAST  
- DAST  
- SBOM  
- dependency scanning  
- CI/CD gates  
- runtime protection
